## バージョン番号の規則
- **MAJOR** (例: v2.0.0): ユーザ体験が大幅に変わる変更の場合
- **MINOR** (例: v1.2.0): 機能追加や大規模なUI変更など
- **PATCH** (例: v1.0.1): バグ修正や小規模なUI修正など
